# alkaidzhen.github.io

1. 工具概述
本工具是一款专业的拧紧工艺统计过程控制（SPC）分析软件，用于评估扭矩和角度参数的过程能力、稳定性及正态性。它集成了：

过程能力指数（CPK、PPK、CMK）计算

8大SPC判异规则自动检测

Jarque-Bera正态性检验及分布形态识别

控制图（含±1σ/2σ/3σ线及规格限）

动态直方图与正态拟合曲线

数据表格（原始数据与分组均值）及异常高亮

中英文双语界面

适用于拧紧工艺开发、批量生产质量监控、设备能力验证等场景。

2. 界面布局
工具界面分为左侧输入区和右侧结果显示区。

左侧输入区
从上至下依次为：

标题与副标题

计算类型选择下拉框

扭矩参数卡片（上公差、下公差、目标值、数据输入框）

角度参数卡片（同上）

全功能计算按钮

三个辅助按钮：清除全部、随机扭矩、随机角度（垂直排列）

右侧结果显示区
标签栏：可切换“扭矩分析”和“角度分析”两个独立视图。

结果卡片区：三个卡片并排（分布与正态性检测、SPC判异、计算结果）。

数据表格区：原始数据表格（可折叠），以及分组数据表格（仅在CPK分组时显示）。

控制图区：包含统计指标行和可交互的控制图。

直方图区：直方图与正态拟合曲线。

3. 输入参数详解
3.1 计算类型
类型	适用场景	标准差计算	推荐样本量
CPK	量产稳定过程	样本标准差（分母n-1）	≥25
PPK	试产初始过程	总体标准差（分母n）	≥25
CMK	设备能力验证	总体标准差（分母n）	≥5
注意：CPK 在数据量≥125且能被5整除时，会自动每5个数据一组取均值，然后基于分组数据计算 CPK、控制图和直方图。原始数据仍显示在表格中。

3.2 公差与目标值
上公差（USL）：规格上限，必须大于下公差。

下公差（LSL）：规格下限。

目标值（Target）：可选，用于计算对中度（(均值-目标)/公差）和最大/最小偏差（(极值-目标)/目标）。若不填，则对中度、最大/最小偏差相关功能不可用，但 CPK 等指数仍正常计算。

3.3 数据输入格式
支持空格、英文逗号、中文逗号、换行作为分隔符。

示例：10.01 10.02 9.98 或 10.01,10.02,9.98。

数据个数至少3个，建议≥10个以获得可靠结果。

3.4 辅助按钮
清除全部：清空扭矩和角度的所有输入框（包括公差、目标值、数据）。

随机扭矩：自动生成符合正态分布的扭矩数据（均值10，标准差0.3），样本量从[5,10,25,50,125]中随机选取，同时自动填入 USL=11、LSL=9、Target=10。

随机角度：类似，生成角度数据（均值90，标准差1.7），自动填入 USL=95、LSL=85、Target=90。

4. 功能操作指南
4.1 基本操作流程
选择计算类型：根据工艺阶段选择 CPK、PPK 或 CMK。

设置规格限：输入上公差、下公差（可选目标值）。

输入数据：粘贴或手动输入扭矩/角度数据。

点击“全功能计算”：等待计算完成（通常<1秒）。

切换标签：查看扭矩和角度的分析结果。

折叠表格：若数据较多，可点击“折叠表格”隐藏原始数据表，再次点击展开。

切换语言：点击右上角“🌐 中/EN”按钮，界面实时切换，已计算数据会重新显示。

4.2 使用随机示例快速体验
点击“随机扭矩” → 自动填充规格和数据 → 点击“全功能计算” → 观察结果。

可多次点击“随机扭矩”以获取不同样本量（5,10,25,50,125）的数据。

4.3 表格交互
原始数据表：每25个数据一个子表，表头为序号和值。异常点（触发SPC规则）背景为浅红色；最大值背景浅绿色；最小值背景浅黄色。

分组数据表（仅当CPK分组时出现）：显示每组均值，同样支持异常高亮和折叠。

折叠按钮：每个表格独立折叠，不影响其他。

4.4 图表交互
控制图：

鼠标悬停在数据点上，会显示该点的数值；若为异常点，额外显示“⚠️ 异常点”。

图中包含：数据点（蓝色圆形，异常点红色圆形，最大/最小值黑色圆形）、均值线（红色虚线）、±1σ/±2σ/±3σ线（灰色虚线）、USL/LSL线（红色实线）。

图表下方统计栏显示均值、UCL、LCL、标准差、中位数、极差、最大值、最小值。

直方图：

动态分组（基于Freedman-Diaconis规则），自动选择最优组距。

蓝色条形为实际频次，红色曲线为理论正态拟合。

5. 结果解读
5.1 分布与正态性检测卡片
项目	含义
Jarque-Bera检验	结论“接受正态”表示p>0.05，数据无显著偏离正态；“拒绝正态”表示p≤0.05。括号内为p值。
分布形态	基于直方图判断：正态、双峰、均匀、正偏、负偏、近似正态。并附有“直方图无明显异常/提示异常”的红色提示。
偏度/峰度	偏度>0表示右偏（右尾长），<0左偏；超额峰度≈0为正态，>0表示尖峰，<0表示平峰。
工程建议：若JB检验拒绝正态，CPK计算结果仅供参考，需结合直方图判断分布是否可接受。

5.2 SPC判异卡片
稳定状态：显示绿色勾和“过程稳定”，表示未触发任何判异规则。

异常状态：显示红色感叹号、“过程异常”，并列出触发的规则列表（红色文字）。底部提示“请检查过程是否存在特殊原因”。

8大判异规则说明：

规则	描述	可能原因
1点超3σ	单点超出控制限	设备故障、材料异常、测量错误
连续7点在均值同侧	过程均值已偏移	刀具磨损、温度变化、操作习惯改变
连续7点单调上升/下降	存在趋势性变化	工具磨损、化学过程、老化
连续14点交替波动	周期性振荡	设备往复运动、轮换操作
3点中有2点超2σ	过程变异增大	材料不均匀、工艺参数波动
5点中有4点超1σ	过程中心可能偏移	同上，但更敏感
连续15点在1σ内	过程变异过小（可能是数据伪造或分层不足）	测量分辨率不足、数据合并不当
连续8点无一在1σ内	过程混合分布	不同批次混入、工具切换
5.3 计算结果卡片
对中度
公式：(均值 - 目标) / 公差 × 100%

颜色：

绿色：|对中度| ≤ 10%

红色：|对中度| > 10% 且为正值

黄色：负值（表示均值低于目标）

提示：下方灰色小字“（均值-目标）/公差，超过±10%提示”。

最大/最小偏差
公式：(极值 - 目标) / 目标 × 100%

颜色：绿色表示|偏差| ≤ 10%，红色表示>10%（无黄色）

注释：下方显示“(相对于目标值的偏差百分比，超过±10%提示)”

过程能力指数
CPK：过程能力，≥1.33合格，≥1.67充足。

PPK：过程性能，判断标准同CPK。

CMK：设备能力，≥1.67合格，≥1.33勉强。

指数旁会显示等级（如“能力充足”、“合格”等）和综合评估（绿色“可接受”或红色“需改进”）。

综合评估条件：

过程稳定（SPC无异常）

能力指数达标（CPK/PPK≥1.33，CMK≥1.67）

对中度|偏差|≤10%

若分布异常，会在评估后附加“（分布异常，仅供参考）”

6. 高级功能
6.1 CPK分组计算
触发条件：计算类型为CPK，且数据量≥125且能被5整除。

效果：

原始数据每5个一组计算平均值，得到新的分析数据集（25组）。

CPK、控制图、直方图、正态性检验等均基于分组均值。

原始数据表格仍显示，分组数据表格额外显示（可折叠）。

目的：消除短周期内随机波动，评估子组间的长期变异。

6.2 中英文切换
点击右上角“🌐 中/EN”按钮，所有界面文字（包括图表标题、卡片内容、按钮文字、表格表头）实时切换。

已计算的数据会自动刷新显示语言。

6.3 表格折叠/展开
每个表格区域（原始数据、分组数据）都有独立的折叠按钮，点击可隐藏/显示表格内容，便于集中查看图表。

6.4 控制图极值标记
最大值和最小值在控制图中用黑色圆形标出，大小6，与正常点（蓝色圆形，大小4）和异常点（红色圆形，大小6）区分。

7. 常见问题与故障排除
Q1: 计算后结果卡片没有数据？
A: 请检查：

公差是否已填且USL > LSL。

数据是否已填（至少3个有效数值）。

计算类型是否正确选择。

目标值未填时，对中度、偏差会显示“无目标值”，但CPK等仍应显示。

Q2: 控制图中出现大量红色点，但SPC卡片显示“未触发”？
A: 红色点仅表示该点在分析数据集中被判定为异常（基于规则），SPC卡片若显示“未触发”则说明所有规则均未触发，两者矛盾。请检查：

控制图的数据源是分析数据集（原始或分组），SPC判异基于同一数据集。

可能存在代码bug，请刷新页面重试。

Q3: 分组数据表格不显示？
A: 分组表格仅在CPK模式且数据量≥125且能被5整除时才会显示。请确认计算类型为CPK，数据个数为125、130、135等5的倍数。

Q4: 直方图显示空白？
A: 可能是数据范围太小或太大导致分组数异常。动态分组算法会限制组数在5~20之间，通常能正常显示。若仍空白，请检查数据是否有NaN或无穷大。

Q5: 移动端表格显示不全？
A: 表格区域支持水平滑动，用手指左右拖动即可。控制图统计栏在小屏上会自动换行。

Q6: 随机数据生成的样本量总是固定的？
A: 每次点击“随机扭矩”或“随机角度”，会从[5,10,25,50,125]中随机选择一个样本量，并非固定。

Q7: 计算按钮点一次后，再点没反应？
A: 程序没有做防抖，但计算是同步的，通常很快完成。如果数据量极大（>10000），可能会卡顿，建议分批输入。

Q8: 中英文切换后，图表标题中的“原始数据”还是中文？
A: 图表标题的动态文本（如“基于原始数据”）会随语言切换更新，请刷新页面确认。若仍有问题，请清除浏览器缓存。

8. 技术规范
8.1 依赖
Chart.js 4.4.8（CDN）

现代浏览器（Chrome 90+, Edge 90+, Firefox 88+, Safari 14+）

8.2 数据精度
所有数值显示保留3位小数（角度数据保留3位，但随机生成时保留1位，表格显示统一3位）。

标准差计算使用无偏估计（分母 n-1）用于CPK，总体标准差（分母 n）用于PPK/CMK。

8.3 性能
支持最多1250个数据点（分块渲染表格，控制图点数量不限，但太多会影响交互）。

直方图分组计算O(n log n)。

8.4 文件信息
单HTML文件，无后端，所有计算在浏览器中完成。

大小约150KB（含样式和脚本）。
